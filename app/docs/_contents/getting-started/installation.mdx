Follow this guide to start using BeeCI and integrate it with your GitHub projects effortlessly.

## Prerequisites

Before you begin, ensure you have the following:

- A GitHub account
- Administrative access to the GitHub repository where you want to install BeeCI

<Note title='Important' type='info'>
  BeeCI requires access to your GitHub repositories. Ensure you have the
  appropriate permissions before proceeding.
</Note>

## Installation Steps

<Stepper>
<StepperItem title="Visit the Landing Page">
Go to the BeeCI landing page:

```plaintext
https://bee-ci.pacia.tech
```

On this page, you'll find a brief introduction to BeeCI's features. Click the **Sign in with GitHub** button to begin.

</StepperItem>

<StepperItem title="Authenticate via GitHub">
After clicking the sign-in button, you'll be redirected to GitHub's authorization page. If you're not logged in, sign in to GitHub or create an account.

- Review the access permissions requested by BeeCI.
- Click **Authorize** to continue.

</StepperItem>

<StepperItem title="Configure BeeCI on Your GitHub Account">
After successful authentication, you'll be redirected to BeeCI's main page. If no repositories appear, you need to install the BeeCI GitHub App:

1. Click the **Configure** button on the main page.
2. Select the account where you want to install BeeCI.
3. Choose specific repositories or grant access to all repositories.
4. Click **Install**.

</StepperItem>

<StepperItem title="Access Your Repositories">
Once installed, return to the BeeCI dashboard. You will see a list of your repositories.

- Use the search bar to find specific repositories.
- Click on a repository to view detailed build logs and status.

</StepperItem>

</Stepper>

## Additional Features

BeeCI includes:

- A comprehensive dashboard summarizing build processes.
- Detailed repository views with build history.
- Integrated documentation accessible from the sidebar menu.

<Note title='Tip' type='success'>
  You can switch between light and dark modes from the sidebar to match your
  preference.
</Note>
